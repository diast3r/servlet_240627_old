clone에 다운받은 프로젝트에 develop 브랜치밖에 없다. 
소스트리에서 원격-origin-master을 가져와야 함.
브랜치 하나만 가져오는 것: check out
origin을 더블클릭하면 체크아웃 창 뜸. 확인 눌러서 체크아웃하면 완료.
master에는 커밋을 안 할 것이기 때문에 굳이 체크아웃 안 해도 됨

******************
집에서 과제를 하는 경우
develop 브랜치에서 코딩하고 커밋 -> 작업한 것 깃허브에 push.
develop 브랜치만 push하면됨.
학원에서 과제(develop에 변경된 사항)을 다운받으려면 pull.

*****************
작업 시작할 땐
변경사항 있을 수도 있으므로 pull부터.
작업 끝나면 develop 브랜치 push.

******************
깃 초기화하려면(없애려면) .git폴더를 지우면 된다.
주의점: sourcetree에서 develop 브랜치를 선택한 상태에서 지워야 작업하던 게 안 날아감.

실수를 해서 커밋을 안 하고 싶을 때:스태시
스태시: 변경사항을 임시로 저장.
스태시에 저장하고 나면 커밋하지 않은 변경사항이 사라지게 됨.
다시 살리고 싶으면 스태시-> 스태시 적용(적용 후 삭제 버튼 체크)

깃허브에 push완료됐는데 pull이 적용이 안 되는 경우
-> pull 됐는데 이클립스에서 표시가 안 되는 경우일 수 있음. 이클립스에서 새로고침해보기.
